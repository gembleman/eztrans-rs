- v3.11 업데이트
사전/필터 또는 파일, 텍스트에 퍼센트가 들어가 있으면 오류가 발생하는 문제 해결

- v3.10 업데이트
UserDic에서 명사/상용어구를 생략하면 사전이 비정상 작동하던 문제 수정

- v3.09 업데이트
UserDic이 읽는 순서대로 우선순위를 정하지 않던 문제가 수정되었습니다
사용자사전 로그에 UserDic_Req, 처리 번호 추가

- v3.08 업데이트
사용자 사전 버그 수정 및 안정화

- v3.07 업데이트
사용자 사전의 일부 단어가 정상적으로 적용되지 않던 문제 수정
사용자 사전의 우선권이 일관적이지 않던 문제 수정. 같은 단어가 발견되면 상용어구, 라인 순 우선
사용자 사전 켜고 끄기 기능 추가
사용자 사전 (UserDic) 처리 내역 추가
/pre, /post 명령어는 사용하더라도 INI에 저장되지 않도록 변경
※ 이번 버전부터 사용자 사전의 단어 속성 최대 길이가 최대 42자에서 36자로 변경되었습니다.

- v3.06 업데이트
사용자 사전 최적화 알고리즘이 정상적으로 동작하지 않던 문제 수정
정규식 엔진을 std에서 boost로 변경했습니다
사용자사전 변환 유틸 jkDicConverter 추가

- v3.05 업데이트
Ehnd가 AppLocale 모드로 열리면 크래쉬가 발생하던 문제 수정
사용자 사전 업데이트시 JKDIC 설정과 관계없이 무조건 UserDict.jk가 로드되던 문제 수정
아네모네 사용중 /reload 명령어 사용시 anedic 파일을 읽지 않는 문제 수정
문자열 유니코드 감시 기능 삭제

- v3.04 업데이트
UserDict에 등록된 단어 길이가 길 경우 로드시 크래쉬가 발생하던 문제 수정

- v3.03 업데이트
Ehnd 설치된 이지트랜스를 VNR이 초기화하지 못하던 문제 수정

- v3.02 업데이트
윈도우 XP 및 일부 PC에서 오류 발생하던 크래쉬 문제 수정
로그 초기 메세지가 출력되지 않던 현상 수정

- v3.01 업데이트
UserDict.jk 파일을 정상적으로 불러오지 않는 문제를 수정
(에그헤드 사용자 사전 또는 아네모네 사용자 사전이 이제 정상적으로 로드됩니다)
로그 비우기 기능이 제대로 작동하지 않던 문제 수정
프로그램 실행시 로그를 비우는 기능 추가 (INI의 FILELOG_STARTUP_CLEAR를 ON으로 했을때 작동)
콘솔에서 일정 줄이 넘어가면 자동으로 비우는 기능 추가 (INI의 CONSOLE_MAXLINE에서 값 조정 가능)

- v3.00 개선사항
사용자 사전 알고리즘의 대폭 향상

프로그램 코드레벨에서 재설계 및 구현
프로그램 경량화 및 속도 최적화
